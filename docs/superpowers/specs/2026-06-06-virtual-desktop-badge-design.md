# fast-blazing-virtual-desktop — дизайн MVP

- **Дата:** 2026-06-06
- **Статус:** утверждён, готов к плану реализации
- **Платформа:** Windows 11 (целевой билд 26200)
- **Язык:** Rust

## 1. Контекст и проблема

Разработчик ведёт несколько фич параллельно. Под каждую заводит отдельный нативный
виртуальный рабочий стол Windows, раскидывает по ним приложения (Chrome, Zed, Figma).
Реальная боль — **ориентация**: при переключении легко потерять, на каком столе
сейчас находишься. Встроенная индикация Windows (имя стола видно лишь в Win+Tab)
для этого слишком пассивна.

Это не задача изоляции приложений и не задача управления окнами. Это задача
**постоянно видимого указателя текущего стола**.

## 2. Цель MVP

Лёгкий фоновый виджет — «бейдж», который **всегда поверх всех окон** и **виден на
каждом виртуальном столе**, показывает имя текущего стола и мгновенно обновляется
при переключении.

### Решение принято: путь A (обёртка над нативными столами)

Строим поверх встроенных виртуальных столов Windows через крейт `winvd`
(обёртка скрытого COM `IVirtualDesktopManagerInternal`). Свой менеджер окон
(show/hide, путь B) не делаем — пользователь уже сидит на нативных столах, надстроить
ориентацию сверху дёшево.

## 3. Не-цели (явно вырезано из MVP)

- CLI-команды управления столами (create / switch / move / rename / pin).
- Запуск приложений на стол и изоляция профилей (`--user-data-dir` и т.п.).
- Шаринг приложения на подмножество столов.
- Полоска-переключатель со всеми столами, выбор стола кликом.
- Глобальные хоткеи переключения.
- Файл конфигурации.

Переименование столов пользователь делает в нативном Win+Tab; бейдж лишь отражает имя.

Эти пункты — кандидаты на v2, не на MVP.

## 4. Архитектура

Один Rust-бинарь, один долгоживущий фоновый процесс. Три внутренних узла:

- **Desktop watcher** (`desktop.rs`) — обёртка над `winvd`. Читает текущий стол и его
  имя, подписывается на события виртуальных столов (смена / переименование / создание /
  удаление).
- **Badge window** (`badge.rs`) — окно Win32: layered, topmost, tool-window,
  click-through. Запинено на все столы. Рисует текст через Direct2D / DirectWrite.
- **Tray** (`tray.rs`) — иконка в системном трее с пунктом «Quit». Холодный путь,
  через крейт `tray-icon`.

### Модель потоков

- **Главный поток** — владелец окна и трея, крутит Win32 message loop.
- **Поток-слушатель `winvd`** — на событие стола вызывает
  `PostMessage(hwnd, WM_APP_DESKTOP_CHANGED, …)`.
- Главный поток на это сообщение перечитывает текущий лейбл и перерисовывает бейдж.

Кросс-поточного рисования (GDI/D2D из чужого потока) нет — вся отрисовка только на
главном потоке.

## 5. Компоненты (модули)

| Модуль | Ответственность |
|--------|-----------------|
| `main.rs` | Старт: инициализация, создание окна, установка трея, пин на все столы, message loop. |
| `desktop.rs` | `winvd`-обёртка: `current_label() -> String`, `spawn_listener(hwnd)`, `pin_self(hwnd)`. |
| `badge.rs` | Регистрация класса окна, создание layered-окна, отрисовка текста, позиционирование (верх-центр, DPI-aware), перерисовка по запросу. |
| `tray.rs` | Иконка трея + меню «Quit». |
| `label.rs` | Чистая функция форматирования лейбла (номер + имя → строка). Единственный легко тестируемый юнит. |

## 6. Поток данных

1. Старт → прочитать текущий стол → отрисовать бейдж → запинить окно на все столы.
2. Пользователь сменил стол (Win+Ctrl+стрелка / Win+Tab) → событие `winvd` →
   `PostMessage` → перерисовка.
3. Переименовал стол в Win+Tab → событие → перерисовка.
4. Создал/удалил стол → номер текущего мог сдвинуться → перерисовка.
5. Quit из трея → unpin, destroy окна, выход.

## 7. Лейбл и визуал

- **Формат лейбла:** есть пользовательское имя → `N · имя` (например `2 · auth`);
  имени нет → `Desktop N`. Номер показываем всегда — переключение Win+Ctrl позиционное.
- **Вид:** скруглённая «пилюля», тёмный полупрозрачный фон, светлый текст.
  Высота ~36px, ширина по содержимому.
- **Позиция:** верх-центр основного монитора, отступ сверху ~8px.
- **Шрифт:** Segoe UI ~15pt, с учётом DPI.
- **Поведение окна:** topmost + click-through, не показывается в таскбаре и Alt-Tab
  (`WS_EX_LAYERED | WS_EX_TRANSPARENT | WS_EX_TOPMOST | WS_EX_TOOLWINDOW`).
- Цвета, позиция, размер, шрифт — константы в коде. Файла конфигурации в MVP нет.

## 8. Обработка ошибок и деградация

- **`winvd` не инициализировался на билде 26200 (риск №0):** показать балун в трее
  «virtual desktop API unsupported», вывести лейбл `Desktop ?`, не падать. План Б при
  неподдержке — перейти на DLL `VirtualDesktopAccessor` или общаться с COM напрямую.
- **Пин на все столы не удался:** бейдж продолжает работать, но виден лишь на одном
  столе; залогировать предупреждение, деградировать мягко.
- **Поток-слушатель событий умер:** включить фолбэк-поллинг текущего стола раз в ~750мс,
  чтобы лейбл оставался актуальным.
- **Смена DPI / монитора (`WM_DPICHANGED`):** пересчитать позицию и масштаб.
- **Сбит topmost** (другое приложение перехватило): переутверждать topmost по таймеру.

## 9. Тестирование

- **Юнит:** `label.rs` — форматирование лейбла (номер + имя → строка), включая случай
  без имени. Чистая функция, покрывается обычными `#[test]`.
- **Спайк-режим `--once`:** запуск печатает текущий лейбл в stdout и выходит, без UI.
  Это де-риск `winvd` на билде 26200. **Реализуется первым.**
- **Ручной чек-лист:** бейдж виден на всех столах; обновляется при переключении;
  click-through пропускает клики; переживает переименование стола; Quit из трея работает.

Win32/`winvd`-код от OS неотделим и автотестами не покрывается — проверяется спайком и
ручным чек-листом.

## 10. Зависимости

| Крейт | Зачем |
|-------|-------|
| `winvd` | Доступ к виртуальным столам: текущий, имя, события, pin окна. |
| `windows` | Win32 API: окно, layered-режим, Direct2D/DirectWrite, DPI, трей. |
| `tray-icon` | Иконка трея и меню «Quit» без ручного `Shell_NotifyIcon`-boilerplate. |
| `anyhow` | Обработка ошибок в путях инициализации. |

## 11. Порядок реализации (де-риск вперёд)

1. **Спайк `--once`** — подтвердить, что `winvd` на 26200 читает текущий стол + имя +
   события + умеет pin. Если нет — включить план Б до продолжения.
2. Layered-окно + отрисовка текста (статичный лейбл).
3. Пин окна на все столы.
4. Подписка на события → перерисовка по `PostMessage`.
5. Трей + Quit.
6. Фолбэки: поллинг при смерти слушателя, переутверждение topmost, обработка DPI.

## 12. Риски

- **`winvd` против билда 26200** — скрытый API мог сдвинуть vtable. Митигация: спайк
  первым шагом; зафиксировать версию `winvd`; план Б (DLL / прямой COM).
- **Topmost против полноэкранных приложений** (игры, полноэкранное видео) — перекроют
  бейдж. Для дев-инструментов приемлемо, не чиним.
- **Дрейф скрытого API между билдами Windows** — пин конкретной версии `winvd`,
  переустановка/проверка после крупных обновлений Windows.

## 13. Будущее (v2, вне scope)

- Полоска со всеми столами и переключением кликом (поглощает «одно место для
  переключения»).
- CLI для скриптов и хоткеев (create / switch / move / pin).
- Запуск приложений на стол, изоляция профилей.
- Автостарт при входе (флаг `--install-autostart` / ярлык в shell:startup).
- Файл конфигурации (позиция, цвета, шрифт).
